## Preface

Next.js has a built-in `next/font` component. Compared with the traditional way of using fonts, using the font component is more flexible and convenient. The use of the font component is mainly divided into two parts, one is Google fonts, and the other is local fonts. Both are implemented through the font component, but the specific configuration is slightly different.

In this article, we will start with the traditional way of using fonts, then explain the convenience and optimization brought by the font component, and finally go into details and explain the specific parameters of the font function. These details only need to be roughly understood when learning, and can be understood in detail when developing actual projects.

Let's talk about the traditional way of using fonts first.

The most basic method is to specify a custom font through `@font-face`. The font file can come from a remote file or a local file. Then use the font in `font-family`.

## 2. next/font and layout offsets

Next.js has a built-in `next/font` component to help you better manage and use fonts. `next/font` will automatically optimize fonts (including custom fonts), for example, using the CSS `size-adjust` attribute to achieve zero layout offset.

We have discussed layout offset in the Image component. In addition to the layout offset caused by not setting the width and height of the image, layout offset is also prone to occur when loading web fonts, for example:

![image.png](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/72115f0d28574811bd7a2ecbc4f9f0b8~tplv-k3u1fbpfcp-jj-mark:3024:0:0:0:q75.awebp#?w=1818&h=626&s=867578&e=png&b=d7dffb)

The `font-size` of the three lines of text is 64px. The only difference is the font. However, if you observe the height of the three lines of text on the left side of the picture, you will find that although the font-size is set to 64px, the actual height is not necessarily 64px. [size-adjust](https://link.juejin.cn/?target=https%3A%2F%2Fdeveloper.mozilla.org%2Fen-US%2Fdocs%2FWeb%2FCSS%2F%40font-face%2Fsize-adjust "https://developer.mozilla.org/en-US/docs/Web/CSS/@font-face/size-adjust") Adjust the size to ensure that the final size is 64px. Next.js automatically makes this optimization.

In addition to preventing layout shift, `next/font` can also help you quickly use Google fonts, and the CSS and font files will be downloaded at build time and saved like other static resources. The browser will not send any requests to Google, ensuring performance and privacy. We will explain more features in detail in this article.

`next/font` is specifically divided into `next/font/google` and `next/font/local`, corresponding to using Google fonts and using local fonts respectively. Let's explain them one by one.

### 3.2. Variable fonts

So what are variable fonts? The so-called variable fonts, quoted from Wikipedia:

> OpenType variable fonts (English: OpenType variable fonts) is an extension specification introduced in the 1.8 version of the font format OpenType. It was jointly developed by Apple, Microsoft, Google and Adobe and officially released on September 14, 2016. Computer fonts that support this specification can store outline variation data and automatically generate rich variation shapes based on the initial glyph outline, allowing users to freely adjust the appearance of the text.

Simply put, variable fonts can freely adjust the width, weight, and tilt of the font, so that one font can display multiple fonts. Next.js recommends using variable fonts.

### 3.3. Specify subset

Google fonts can specify subsets（[subset](https://link.juejin.cn/?target=https%3A%2F%2Ffonts.google.com%2Fknowledge%2Fglossary%2Fsubsetting "https://fonts.google.com/knowledge/glossary/subsetting")），as [Roboto Mono](https://link.juejin.cn/?target=https%3A%2F%2Ffonts.google.com%2Fspecimen%2FRoboto%2BMono%3Fhl%3Dzh-cn "https://fonts.google.com/specimen/Roboto+Mono?hl=zh-cn") Support Latin, Cyrillic, Greek, etc. We don't need to use all of them, so we can use the `subsets` attribute to specify a subset, which can also reduce the size of the font file and improve performance.

These subsets will be preloaded by default (controlled by the `preload` attribute, which will be discussed later in this article). If `preload` is `true`, but no subset is specified, there will be a warning. Some fonts have only one default subset, such as `latin`, which also needs to be manually specified.

> Latin letters, also known as Roman letters, refer to a mature alphabet system based on the ancient Roman letters. They were originally circulated in the Italian peninsula and Western Europe. In the 19th century, they spread to become the most popular letters in the world. They are also the most widely used letters in the world. They are the only standard fonts in most modern European and American countries. Latin letters are the 26 letters we write. So many font subsets have latin.

### 5.1. src parameter

It must be passed in `next/font/local`, which can be a string or an object array (type: `Array<{path: string, weight?: string, style?: string}>`). The path address is equivalent to the location where the font loading function is called.

For example, in `app/page.js`, `src:'./fonts/my-font.woff2'` is used to call the font loading function, and `my-font.woff2` is placed under `app/fonts/`.

### 5.2. weight

Values ​​can be either strings like `weight: '400'`, `weight: '100 900'` (variable fonts, range from 100 to 900), or arrays of strings like `weight: ['100','400','900']` (3 possible values ​​for immutable fonts).

### 5.3. style

Concept Reference [font-style](https://link.juejin.cn/?target=https%3A%2F%2Fdeveloper.mozilla.org%2Fzh-CN%2Fdocs%2FWeb%2FCSS%2Ffont-style "https://developer.mozilla.org/zh-CN/docs/Web/CSS/font-style")，The default value is `normal`, other values ​​include `italic`, `oblique`, etc., see[font-style](https://link.juejin.cn/?target=https%3A%2F%2Fdeveloper.mozilla.org%2Fzh-CN%2Fdocs%2FWeb%2FCSS%2Ffont-style%23values "https://developer.mozilla.org/zh-CN/docs/Web/CSS/font-style#values")。If using a non-variable font from `next/font/google`, you can also pass in an array of style values, such as `style: ['italic','normal']`.

### 5.6 display

Concept Reference [font-display](https://link.juejin.cn/?target=https%3A%2F%2Fdeveloper.mozilla.org%2Fzh-CN%2Fdocs%2FWeb%2FCSS%2F%40font-face%2Ffont-display "https://developer.mozilla.org/zh-CN/docs/Web/CSS/@font-face/font-display")。By default, when we load a font, the place where the font is used will be blank at first, and then it will not be displayed until the font is downloaded. CSS font-display controls this process. The values ​​of CSS font-display are `'auto'`, `'block'`, `'swap'`, `'fallback'`, `'optional'`, which are the same as the display values ​​of the `next/font` component. Here are the differences in the loading effects of these values:

1. **auto**: Use the browser's default value, usually block
2. **block**: Display blank (3s) first, then switch to the backup font, and switch after the font is loaded
3. **swap**: Display the backup font first, and switch after the font is loaded
4. **fallback**: Display blank (about 100ms) first, then switch to the backup font, the time is about 3s, if the font can be loaded within 3s, use the font, if it cannot be loaded within 3s, then use the backup font
5. **optional**: Display blank (about 100ms) first, if it can be loaded within 100ms, use it, if it cannot be loaded, directly use the backup font

The default value of CSS `font-display` is `auto`, and the default value of `next/font` component is `swap`.

### 5.7. preload

Boolean value, specifies whether fonts should be preloaded, default value is `true`.

### 5.8. fallback

The fallback font when the font cannot be loaded. There is no default value. It is in the form of a string array, such as `fallback: ['system-ui', 'arial']`。

### 5.9. adjustFontFallback

For `next/font/google`, `adjustFontFallback` is a boolean value that sets whether the automatic fallback font should be used to reduce cumulative layout shift. The default value is `true`.

For `next/font/local`, `adjustFontFallback` can be a string or `false`. Possible values ​​are `Arial`, `Times New Roman`, `false`. The default value is `Arial`.

`Arail` is a classic sans serif font, and `Times New Roman` is a classic serif font. In fact, it allows you to choose a serif or sans serif font as a fallback font for adjustment. Of course, you can also choose not to adjust, that is `false`. The reason why you don't need to choose Google fonts is because Next.js automatically judges it for you.

### 5.10. variable

This property is related to CSS variables. Let's briefly review the concept of CSS variables.

CSS variables are defined by the developer themselves. First, the value is set through the custom property tag (for example: `--main-color: black;`), and then the value is obtained by the `var()` function (for example: `color: var(--main-color);`).

There are two benefits. One is that it is easy to reuse. For example, a color value may be used in multiple places. If it changes, you need to search and replace it globally. With CSS variables, you only need to change the value of the variable. The second is semantics. For example, `--main-text-color` is easier to understand than `#00ff00`. Custom properties are subject to cascading and inherit their values ​​from their parents.

### 7.3. Using font definition files

Every time a font function is called, the font will be hosted as an instance, so if the same font is used in multiple places, it should still be loaded in one place and then imported on demand. This is the role of font definition files.

For example, create a `fonts.ts` file in the `styles` folder in the root directory, and then declare the font definition:
